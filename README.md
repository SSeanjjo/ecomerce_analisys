## 📖 Project Overview

This project performs a full end-to-end analysis of a real-world e-commerce dataset covering **541,000+ transactions** from a UK-based online retailer. The analysis spans the complete data analytics lifecycle: ingestion, cleaning, exploratory analysis, statistical testing, feature engineering, and interactive dashboarding.

The project is designed to be **fully reproducible**: clone the repo, follow the setup steps, and every result can be regenerated from scratch.

**Business context:** The retailer sells unique all-occasion giftware. Most customers are wholesalers. The goal is to understand revenue drivers, customer behavior, and the effectiveness of discount strategies.

## 📊 KPI Definitions

### Revenue KPIs

| KPI | Definition | Formula |
|-----|-----------|---------|
| **Total Revenue** | Sum of all completed order values | `SUM(Quantity × UnitPrice)` where Quantity > 0 |
| **Average Order Value (AOV)** | Mean revenue per transaction | `Total Revenue / Number of Orders` |
| **Revenue Growth Rate** | Month-over-month revenue change | `(Current Month Revenue - Prior Month Revenue) / Prior Month Revenue × 100` |
| **Revenue by Category** | Total revenue grouped by product category | `SUM(Revenue) GROUP BY Category` |

### Customer KPIs

| KPI | Definition | Formula |
|-----|-----------|---------|
| **Active Customers** | Unique customers with at least one order in the period | `COUNT(DISTINCT CustomerID)` |
| **Customer Retention Rate** | % of customers who return in the next month | `Returning Customers / Prior Month Customers × 100` |
| **Churn Rate** | % of customers who did NOT return | `1 - Retention Rate` |
| **Customer Lifetime Value (CLV)** | Estimated total revenue from a customer | `AOV × Purchase Frequency × Average Customer Lifespan` |

### RFM KPIs

| KPI | Definition |
|-----|-----------|
| **Recency (R)** | Days since the customer's last purchase |
| **Frequency (F)** | Total number of orders placed by the customer |
| **Monetary (M)** | Total revenue generated by the customer |
| **RFM Score** | Composite score (1–5 each) used to classify customer segments |

### RFM Segments

| Segment | RFM Score Range | Description |
|---------|----------------|-------------|
| Champions | R=5, F=5, M=5 | Bought recently, buy often, spend the most |
| Loyal Customers | F≥4, M≥4 | Regular buyers with high value |
| At Risk | R≤2, F≥3 | Used to buy often but haven't recently |
| Lost | R=1, F=1 | Last purchase was very long ago |
| New Customers | R=5, F=1 | Bought recently but only once |

### Operational KPIs

| KPI | Definition |
|-----|-----------|
| **Return Rate** | % of transactions with negative quantity (returns) |
| **Orders per Customer** | Average number of orders placed per unique customer |
| **Top 10 Products** | Products ranked by total quantity sold |

## ⚙️ Setup Instructions

### Prerequisites

- [Anaconda](https://www.anaconda.com/download) (Python 3.11+)
- [Git](https://git-scm.com/)
- [Power BI Desktop](https://powerbi.microsoft.com/desktop/) (Windows only, free)
- [Tableau Public](https://public.tableau.com/en-us/s/download) (free)
- A free [Kaggle account](https://www.kaggle.com/) for dataset download

---

### 1. Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/ecommerce-analysis.git
cd ecommerce-analysis
```

### 2. Create and activate the Conda environment

```bash
conda create --name ecommerce_analysis python=3.11 -y
conda activate ecommerce_analysis
```

### 3. Install dependencies

```bash
conda install -c conda-forge -y numpy pandas matplotlib seaborn scipy statsmodels scikit-learn jupyterlab ipykernel openpyxl sqlalchemy psycopg2

pip install kaggle plotly nbformat nbconvert
```

### 4. Register the Jupyter kernel

```bash
python -m ipykernel install --user --name ecommerce_analysis --display-name "E-Commerce Analysis"
```

### 5. Configure Kaggle API

1. Go to [kaggle.com](https://www.kaggle.com) → Account → Settings → API → **Create New Token**
2. A `kaggle.json` file will download
3. Place it in:
   - **Windows:** `C:\Users\YOUR_NAME\.kaggle\kaggle.json`
   - **Mac/Linux:** `~/.kaggle/kaggle.json`

### 6. Download the dataset

```bash
kaggle datasets download -d carrie1/ecommerce-data -p data/raw/ --unzip
```

### 7. Launch JupyterLab

```bash
jupyter lab
```

Open the notebooks in order starting with `notebooks/01_eda.ipynb`. Make sure to select the **"E-Commerce Analysis"** kernel.

### 8. Export environment (for reproducibility)

After installing everything, save your environment so others can replicate it exactly:

```bash
conda env export > environment.yml
```

To recreate from this file on another machine:

```bash
conda env create -f environment.yml
conda activate ecommerce_analysis
```

## 📂 Data Sources

| Dataset | Source | Records | Link |
|---------|--------|---------|------|
| UK E-Commerce (primary) | Kaggle | ~541,000 | [carrie1/ecommerce-data](https://www.kaggle.com/datasets/carrie1/ecommerce-data) |

| Online Retail II (UCI) | UCI ML Repository | ~1,000,000 | [UCI Dataset 502](https://archive.ics.uci.edu/dataset/502/online+retail+ii) |

### Data Dictionary (UK E-Commerce Dataset)

| Column | Type | Description |
|--------|------|-------------|
| `InvoiceNo` | String | Unique invoice number. Prefix `C` = cancellation |
| `StockCode` | String | Unique product code |
| `Description` | String | Product name |
| `Quantity` | Integer | Units per transaction. Negative = return |
| `InvoiceDate` | DateTime | Date and time of transaction |
| `UnitPrice` | Float | Price per unit in GBP (£) |
| `CustomerID` | Float | Unique customer identifier (nullable) |
| `Country` | String | Country of customer residence |

> **Note:** Raw data files are excluded from version control via `.gitignore`. Download them using the Kaggle API command in Step 6 above.
